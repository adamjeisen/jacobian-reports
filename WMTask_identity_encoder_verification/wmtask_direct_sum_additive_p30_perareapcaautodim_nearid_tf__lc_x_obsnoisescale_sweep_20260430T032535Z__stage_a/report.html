Experiment Context

wmtask_direct_sum_additive_p30_perareapcaautodim_nearid_tf__lc_x_obsnoisescale_sweep

Description

WMTask fully-observed (N1=N2=64), latent JacobianODE with DirectSumCouplingEncoder, PCA-based per-area autodim (n_target_var_threshold=0.99). Each area independently picks its own n_target_dims as the smallest k such that the first k PCs capture >= 99% of that area's variance; remaining dims become a per-area null subspace. Additive coupling, 8 layers, hidden_dim=128. near_identity_std=1e-3, final_perm_identity=true. 21-cell sweep over 7 LC x 3 obs_noise_scale. Split-mode loss. TF-coupled LR schedule (k_scale=1, init 1e-4 -> min 1e-6 following TF alpha annealing). Two-stage protocol with dual-checkpoint (primary ES patience=5, shadow patience=2).

Hypothesis

Companion to the full-128 DirectSum sweep with the same TF-coupled LR + near-id init recipe. The per-area null subspace gives loop closure something structural to clamp (vs full-128 where z_dyn is the entire latent and null is empty); PCA-99% picks a smaller per-area n_target_dims (likely on the order of 10-30 per area for the WMTask RNN's effective rank). If the smaller dim helps dynamics learning AND the null subspace + LC pressure is doing useful regularization, this should match or beat full-128 on val traj_loss while preserving cross-area Gramian asymmetry. Replaces the FNN-autodim variant which produced unusable per-area dim estimates on this data.
